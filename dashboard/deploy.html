Deploy your AgentLens instance

Run AgentLens on your own infrastructure. Your LLM data never leaves your network. This page only stores deployment metadata (label, registered date) — never your traces.

Air-gap mode is enabled by default. The container makes zero outbound calls to Anthropic, Stripe, Resend, or any third-party. Verify with tcpdump -i any 'not port 8000' after install.

1Run the installer on your Linux host

One command. Pulls a smoke-tested image from GitHub Container Registry, generates a strong admin token locally, starts the container, and prints your tenant API key.

curl -fsSL https://www.agentlens.one/install | bash

Want to pin a label or port? Pass via env:

AGENTLENS_LABEL=my-workload AGENTLENS_PORT=8000 \ curl -fsSL https://www.agentlens.one/install | bash

2Register this deployment

Lets us track that your instance was deployed (for pilot support, not data access). Register one entry per host you run AgentLens on.

3Add to your code

Use the API key the installer printed at the end of step 1 (not the agentlens.one portal key — they're separate by design).

import agentlens agentlens.init( api_url="http://<your-host>:8000/ingest", api_key="al_xxx_from_installer_output", ) agentlens.patch_openai() # also covers Ollama (OpenAI-compatible)

For multi-step workflows (GraphRAG, reasoning chains), use trace_agent():

from agentlens import trace_agent with trace_agent("my_workload_query", input=user_question) as t: with t.span("retrieve", span_type="retrieval") as s: s.set_output(f"{len(passages)} passages") with t.span("synthesize", span_type="llm", model="llama3.1") as s: s.set_output(answer) s.set_tokens(usage.total_tokens) t.set_output(answer)
